Prompt Make

一款面向普通用户的浏览器插件：在 ChatGPT、Kimi、Claude 等输入框旁，一键把模糊想法优化成可直接发送给 AI 的高质量 Prompt。

项目类型 Chrome 插件 / AI Prompt 工具 / MVP
我的角色 产品策略、PRD、交互设计、AI 规则设计、vibe coding
目标用户 AI 新手、运营、电商店铺后台用户、产品、设计师、办公人群
核心价值 降低提问门槛，提高 AI 输出质量，形成 Prompt 复用资产

这个产品重点展示的不是“做了一个插件”，而是如何把 AI 能力封装为低门槛、可验证、可商业化的产品体验。

产品解决的痛点：用户知道 AI 有用，但不知道怎么问。

普通用户的真实输入往往是一句话想法，缺少角色、目标、背景、约束和输出格式。Prompt Make 的亮点是把专业 Prompt 结构藏在后台，把前台体验压缩为“输入框旁一键优化”。

提问质量不稳定

用户常把任务说得过短，导致 AI 输出泛泛而谈。产品通过场景规则补齐角色、目标、限制、格式和质量标准。

工具切换成本高

用户不愿为了优化 Prompt 再打开一个新网站。插件形态让优化动作发生在用户正在输入的地方。

把复杂变成一次点击

核心链路是“输入内容 → 点击优化 → 替换回输入框”，用最短路径验证 AI 工具价值。

产品设计说明：用业务流程图讲清楚从用户输入到商业转化的链路。

作品集不能只放动效。这里用静态展板讲清楚产品判断、业务流程、为何选择浏览器插件，以及插件核心界面。

为何选择浏览器插件
更贴近真实场景用户使用 AI 的入口已经在网页输入框里，插件可以自然贴近输入行为。
学习成本更低不需要进入复杂工作台，不需要理解 Prompt 模板体系。
跨平台覆盖更广无需分别适配单一 AI 产品账号体系，可在多个网页输入框中复用同一套优化能力。

应用场景 Prompt 规则：把专家思考拆成可执行模板。

规则层不是简单改写，而是针对不同任务补齐必要上下文。前台只显示场景标签，后台用系统提示词控制优化方向。

设定思路先覆盖普通用户最常见的 AI 使用任务，再保留与你过往电商和后台系统经验相关的专业场景。
为何是这 6 个场景通用解决长尾需求；自媒体、电商、办公是高频刚需；产品、设计师能体现你转 AI 产品经理的职业方向。
规则设计原则每个场景都围绕角色、目标、背景、约束、输出格式、质量标准展开，但侧重点不同，避免模板千篇一律。
通用 角色、目标、背景、约束、输出格式、质量标准。

适合大多数模糊提问，把一句话改成清晰任务说明。

自媒体写作 平台、人设、读者、标题、正文结构、标签、互动引导。

适合小红书图文、短视频脚本、种草内容和选题策划。

电商 商品信息、目标用户、卖点、平台语气、转化目标、禁用表达。

适合详情页、活动文案、客服话术、店铺后台运营任务。

产品 用户、场景、痛点、目标、功能范围、验收标准、风险。

适合 PRD、竞品分析、功能拆解、需求澄清。

设计师 页面目标、用户路径、视觉风格、组件状态、交互反馈、适配要求。

适合页面设计、组件说明、交互稿描述和设计评审准备。

办公协作 沟通对象、背景、责任人、时间节点、风险、下一步行动。

适合邮件、会议纪要、任务拆解、周报和跨部门沟通。

技术实现架构：前端插件 + DeepSeek API + 场景规则层。

MVP 选择零构建 Chrome 插件，快速跑通“网页输入框识别、悬浮面板、模型调用、结果替换、历史收藏、试用次数”这些关键链路。

内容脚本识别文本框和可编辑区域，渲染优化按钮和悬浮面板。
插件弹窗控制功能开关、场景选择、月度试用次数提示。
后台服务接收优化请求，拼接系统提示词，调用模型接口。
模型接口负责生成优化后的 Prompt，输出简体中文可复制文本。
本地存储保存设置、历史记录、收藏和试用次数。
AI 用在优化生成模型不执行用户任务，而是优化用户 Prompt 本身，确保输出可以直接复制给 AI 使用。
AI 用在场景增强通过系统提示词约束不同场景的补全方向，例如电商强调卖点和转化，产品强调验收标准。
AI 用在质量控制规则要求保留原意、不编造事实、变量逐字保留、信息不足时提示补充。
